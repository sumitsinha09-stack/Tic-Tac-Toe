Why Minimax?

Minimax is the standard algorithm for two-player zero-sum games. It operates on one core assumption:

Both players play optimally. The maximizer (X) picks the highest-scoring move. The minimizer (O) picks the lowest. Neither makes mistakes.
  • X wins → +10 (minus depth for speed preference)
  • O wins → −10 (plus depth for delayed loss)
  • Draw → 0

Minimax Algorithm

The core recursive function that powers the AI.

function minimax(board, depth, isMaximizing):
  // Base cases — terminal states
  if X wins: return +10 - depth
  if O wins: return -10 + depth
  if board full: return 0

  if isMaximizing: // X's turn
    bestScore ← -∞
    for each empty cell:
      place X
      score ← minimax(board, depth+1, false)
      undo move
      bestScore ← max(bestScore, score)
    return bestScore
  else: // O's turn
    bestScore ← +∞
    for each empty cell:
      place O
      score ← minimax(board, depth+1, true)
      undo move
      bestScore ← min(bestScore, score)
    return bestScore

Alpha-Beta Pruning

α (alpha) = best score the maximizer can guarantee
β (beta) = best score the minimizer can guarantee
If α ≥ β, prune remaining branches — they cannot change the outcome.
function alphabeta(board, depth, α, β, isMax):
  if terminal: return evaluate(board)
  if isMax:
    val ← -∞
    for each move:
      val ← max(val, alphabeta(..., α, β, false))
      α ← max(α, val)
      if α ≥ β: break // β cutoff
    return val
  else:
    val ← +∞
    for each move:
      val ← min(val, alphabeta(..., α, β, true))
      β ← min(β, val)
      if α ≥ β: break // α cutoff
    return val

Key Takeaway

With optimal play from both sides, Tic Tac Toe always ends in a draw. Minimax guarantees this — making the AI unbeatable.

This simple 3×3 grid remains one of the best teaching tools in CS — a gateway to adversarial search, game theory, and the AI techniques powering modern engines for Chess, Go, and beyond.
